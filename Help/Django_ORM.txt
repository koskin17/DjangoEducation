Команды в оболочке django shell:
Запуск оболочки: python manage.py shell

Сначала импортитруются модели, с которыми будем работать;
Получить все данные: News.objects.all();
Измение метода сортировки: Изначально сортировка не определена, но в модели она может быть прописана и тогда будет выводиться с соответствующе сортировкой. Также сортировку можно можно измениить непосредственно в запросе: News.objects.order_by('pk')
    Использование 'pk' - это универсальный способ, которые позволяет не запоминать как был назван первичный ключ. Django сама определяет этот первичный ключ (primaty key) и выполненяет с ним необходимые действия.
Также есть метод reverse() - News.objects.all().reverse()
Для получения одной записи используется метод get() - News.objects.get(pk=1), News.objects.get(title='News 5'). С методом get() нужно быть аккуратным - если в БД будет 2 и более записей, то django при спользовании метода get() вернёт ошибку. По этой причине метод get() используется для поиска по ключевым параметрам: pk или slug.

При получеи объекта django загружает все его поля, т.е. все аттрибуты модели у объекта доступны для работы после получения объекта.
При изменениии метода __str__ в модели и запросе к аттрибуту будет выводиться строка, которая описана в методе __str__
Также можно получить данные из связанных данных, например primary key категории, которая присвоена новость: news1.category.pk
